Live capturing of DASH7 traffic in wireshark

Date Thu 28 August 2014 Tags Tools

As promised in the previous post we will show how to do live capturing in wireshark instead of reading from a previous captured file. Instead of dumping the PCAP data to a regular file the logger writes the data to a named pipe and wireshark reads from this named pipe. As a prerequisite, make sure the setup of the previous post is functional, since we will be building on top of this.

When the d7-oss-logger.py script is started with the -p parameter it will dump the received traffic to a named pipe. The filename of this named pipe depends on the OS you are running, for POSIX systems this is /tmp/oss7-pipe, for MS Windows this is \\.\pipe\oss7-pipe. The logger will create the named pipe and wait for a client connection (by wireshark in our case) before it writes data to it. When wireshark conencts the logger will first transmit the PCAP header, followed by the subsequent frames which are received. It is possible to stop listening in wireshark and reconnect later.

Now, how to configure wireshark to use this named pipe? We add the named pipe 'interface' by choosing Capture > Options ... > Manage Interfaces > New and filling in the pipe filename mentioned above in the Pipe field. The named pipe should appear in the interface list, and you can now start listening on it.

One caveat to keep in mind is that when you quit the logger while capturing in wireshark you will need to restart the capture in wireshark again after restarting the logger. Being able to start the logger from within wiresharj would make the integration easier, but this is not possible in wireshark currently. However, there is a proposal which might enable this in the future.

Happy capturing!
