27 /* \brief Implementation of hw_gpio_configure_pin
28  *
29  * This function supports all pin configuration modes available through the
30  * 'GPIO_PinModeSet' function in emlib.
31  *
32  * It should be noted that the EZR32WG only has a single interrupt bit for each pin id
33  * and that only a single GPIO port can be selected for each pin id. This means that it is,
34  * for instance not possible to configure an interrupt both for in A3 and D3 since these pins share
35  * the same pin id (3 in this case) but are wired on different ports. A3 and D4 do not conflict
36  * because they have a different pin id.
37  *
38  * Because of this, this function also requires the user to explicitly choose whether or not
39  * interrupts can be enabled for a specific pin through the <int_allowed> parameter.
40  * If <int_allowed> is true for the given pin, the interrupts can later be configured
41  * through 'hw_gpio_configure_interrupt'. Otherwise, 'hw_gpio_configure_interrupt' will
42  * always return EOFF for the given pin.
43  *
44  * When this function is called with <int_allowed> set to true, after interrupts have been
45  * explicitly allowed for a conflicting pin, this function will return EBUSY and the gpio port
46  * will not be configured.
47  *
48  * \param pin_id(pin_id_t) The pin_id of the GPIO pin to configure
49  * \param int_allowed(bool) Whether or not to allow interrupts for this pin
50  * \param mode(GPIO_Mode_TypeDef) The mode for the GPIO pin (see EZR32WG docs for more information)
51  * \param out(unsigned int) The out value for the GPIO pin (see EZR32WG docs for more information)
52  *
53  * \return SUCCESS if the pin was configured successfully
54  * EALREADY if the pin was already configured through a previous call to this function
55  * EBUSY if <int_allowed> is true and interrupts were already allowed for a conflicting pin
56  */
